### Лекция №1 по 3D Моделированию

Рассказали про растр.
Про то, как рисовать фигуру на осциллографе.

`RGB` - Red, Green, Blue.
`CMYK` - Cyan, Magenta, Yellow, Black (Вычитающийся)

Теперь отрезок на растре. На доске была сеточка. Узлы = пиксели.
![[Pasted image 20250905194706.png]]
Порисуем точки, как по координатам `y = kx + b`
k = (y1-y0)/(x1-x0)
b = y0 - kx0

| x0     | y0                   |
| ------ | -------------------- |
| x0 + 1 | round(k(x0 + 1) + b) |
| x0 + 2 | round(k(x0 + 2) + b) |
|        |                      |
``` go
for _, x := range nums {
	y = round(kx + b)
	fmt.Print(x, y)
}
```
Но это долго!!! `round` для float. А если у нас миллион отрезков?
Отсюда пошло развитие Демосцен (Demoscene). Наиболее красиво с малыми затратами.

А давайте увеличим масштаб раз так в 100
`(x1-x0)y =k(x1-x0)+d(x1-x0)`

Новые точки можно поставить на той же горизонтали или выше на одну.
`x += 1`
Если диагональ, то `y = y +1`
Иначе - `y = y`

На каждом шаге у нас увеличивается `y` на некую `delta = k`.
Если у нас поднятие превышает 0.5, то ставим выше, иначе - на уровне. 
(`nk - y > 0.5`)
(`nk - y <= 0.5`)
`n` хранить не в кайф, давайте уберём её, тем что на каждом шаге будем менять переменную. Было `0 -> 0.6 -> 1.2 -> 1.8 -> 2.4` стало `0 -> 0.6 -> -0.4 -> 0 -> 0.6 -> -0.4`
Введём `ε`, которая находится в интервале [0.5, 0,5] 
`ε = ε + k`, если `ε` > 0.5, то `ε = ε - 1`

Итого нашем шагом
``` go
x = x + 1;
ε = ε + k
if ε > 0.5 {
	y = y + 1
	ε = ε - 1
} else {
	y = y
}
```
А теперь хотим избавиться от чисел с плавающих точек при помощи масштабирования. Вспомним, чему равняется `k`

Итого получим **Алгоритм Брезенхема** ![[LineBresenham.gif]]

``` 
ε' = 2ε(x1-x0) = 0
x = x + 1
ε' = ε' + 2(y1 - y0)
if ε' > (x1 - x0) {
	y = y + 1
	ε' = ε' - 2(x1 - x0)
} else {
	y = y
}
```

**Метод Монте Карло.**
Площадь круга. Накидываем рандомно точки. Посчитать, сколько их.
А что если точки равноудалены, то утолщение.
Будем считать, что только одна ордината для одного абсцисса.
