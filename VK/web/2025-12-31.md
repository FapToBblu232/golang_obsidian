Интернет - сеть, а www - множество ресурсов, связанные вместе
Каждый уровень - свой функционал
- Физический - работа с байтами. А как вообще передать данные в нашем мире? Нам не важно, какие именно данные.
- Канальный - работа с кадрами. Отправка данных между устройствами
- Сетевой - IP - Internet Protocol, маршрутизация, адресация. Как в рамках глобальной сети отправить пакет. Мы не запариваемся, как целостно придёт, только куда придёт. Не гарантируем, дойдёт ли до адресанта.

#### TCP/IP
Семейство протоколов Интернета и модель из 4х слоёв, обеспечивающих адресацию, маршрутизацию и передачу данных между узлами сети.
- **В IPv4:** 4 октета (8 бит) = 32 бита. (2^32 = 4 млрд)
- **В IPv6:** 8 гекстетов (16 бит) = 128 бит (16 байт).
**NAT** (Network Adress Translation) - можем скрыть всю сеть под одним IPшником. А внутри маршрутизируем закрыто от всей внешней сети.
**VIP** - виртуальные
Переход к IPv6 - сложный

#### Транспортный
Оперируем порами.

**TCP** (Transmission Control Protocol) - Надёжный с Three way handshake процессом установки соединения. Готовы ли отсылать + принимать данные?
**UDP** (User Datagram Protocol) - другой транспортный протокол, отправляет дейтаграммы без гарантий доставки, порядка и повторной передачи. Минимальная задержка и накладные расходы.

Проблема TCP заключается в Head of line bloking (**HOL**) - задержка всей очереди из-за блокировки её головного элемента. Один не дошёл -> тормоза + лаги, потому в стримминге и играх TCP вообще не используются.

#### World Wide Web
WWW - распределенная гипертекстовая система поверх Интернета, в
которой документы, связанные ссылками и адресуемые через URL,
передаются по стандартным протоколам (прежде всего HTTP/HTTPS)
между веб-серверами и клиентами (браузерами).

WWW - набор протоколов, серверного и клиентского ПО,
позволяющих получать доступ к документам

По своей сути - огромный граф из гиперссылок.

#### Типы документов (MIME-типы/Multipurpose Internet Mail Extensions)
Список MIME-типов:
- text/html
- text/css
- text/javascript
- image/png
- video/mp4
- text/xml
- application/json
Расширения файлов играют второстепенную роль
В http за передачу MIME типов отвечает заголовок **Content-Type.**

**JSON** - Java Script Object Notation - удобно для JS frontend разраба
**XML** имеет следующие минусы:
- Json лучше по кол-ву символов, но при сжатии роли не играет
- Сериализация. В XML один объект может быть представлен разными вариантами, когда как в JSON Ключ-Значение

#### Виды документов
**CDN** (Content Delivery Network) - геораспределённая сеть, которая за счёт физического расстояния позволяет быстрее отдавать данные.
- Статические
	- Это файлы на дисках сервера
	- Как правило, обладают постоянным адресом

- Динамические
	- Создаются на каждый запрос
	- Содержимое зависит от времени и пользователя
	- Адрес может быть постоянным или меняться

#### URL vs URI
Гораздо приятнее работать, когда существует единый формат.
**URI** (Uniform Resource Identifier) - общий идентификатор ресурса. Это можно встретить в книгах.
**URL** (Uniform Resource Locator) - частный случай URI, который указывает, как
найти ресурс. Частный случай для веба.

Из каких частей???
`http://server.org:8080/path/doc.html?a=1&b=2#part1`
- http - протокол
- server.org - DNS имя сервера
- 8080 - ТСР порт
	- 80 - стандарт для http
	- 443 - https
- /path/doc.html - путь к файлу. не в файловой системе. Удобная абстракция.
- a=1&b=2 - опции запроса. query параметры.
- part1 - якорь, положение на странице. **Бэкенд его не видит.**

Существуют URL:
- Абсолютные
- Относительные 

#### Виды архитектур информационных систем
- Client-Server.  Сервер предоставляет **API** (Application Programming Interface) - контракт. По определённому набору информации отдаст что-то.
	- Client - На компах конечных пользователей. Задача - получение и отображение документов. ВОт тут вот не запариваются по поводу того, как отдаются файлы. Взаимодействуем по APIшке и славно.
	- Сервера - хранение и отдача.
- File-Server. Отдаём клиенту весь файл, чтобы юзер сам распарсил и работал с этим файлом.
- Peer-to-Peer. Без посредников. Торрент. 

#### Браузер
Пользовательский агент. 
- по URI (обычно HTТР/HTTPS) извлекает представления ресурсов;
- парсит и интерпретирует форматы (HTML, CSS, JS и др.), формирует DOM/стек контекстов просмотра;
- исполняет сценарии и события, управляет состоянием (history, storage, cookies), безопасностью (origin, CSP, sandbox);
- рендерит результат и посредничает во взаимодействии пользователя с ресурсами.

**SPA** (Single Page Application) - не запрашиваем новые данные, кроме json'a.
**UI** (User Interface)
**DOM** (Document Object Model) - Древовидное представление структур данных. HTML/XML. V8 занимается парсингом в это. Отдаст это рендерингу, а дальше оно показывается пользователю
